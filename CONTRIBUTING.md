# Contributing to KeRo AI Prompt Library

Thanks for your interest in contributing.

You can contribute by:

- Adding new prompts
- Improving existing prompts
- Adding examples
- Translating prompts
- Fixing structure or formatting
- Reporting low-quality or outdated prompts

## Prompt Quality Standard

Each prompt should include:

- Clear use case
- Input variables
- Prompt body
- Example input
- Expected output
- Notes when needed

## Prompt Format

Use [docs/PROMPT_TEMPLATE.md](docs/PROMPT_TEMPLATE.md) when adding or standardizing a prompt.

Variables should use the `{variable_name}` format, for example:

- `{product_name}`
- `{target_audience}`
- `{platform}`
- `{tone}`
- `{language}`

## Pull Request Checklist

- [ ] The prompt has a clear use case.
- [ ] Variables are clearly defined.
- [ ] The prompt is formatted properly.
- [ ] An example is included when useful.
- [ ] No private or sensitive data is included.
- [ ] Product claims, prices, certifications, and platform rules are not invented.
- [ ] Markdown code blocks are closed.
